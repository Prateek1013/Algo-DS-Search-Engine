
Given two arrays AA and BB of length NN each. 
You can do the following operation on the array BB exactly once:

Chose two integers RR (1≤R≤N)(1≤R≤N) and K(K≥0)K(K≥0) and apply KK right-rotations to the subarray [1,R][1,R].
Applying a right-rotation to the array C=[C1,C2,C3,…,CM]C=[C1,C2,C3,…,CM] changes it to [CM,C1,C2,…,C(M−1)][CM,C1,C2,…,C(M−1)] 

Find whether you can transform the array BB into the array AA by applying the mentioned operation exactly once. If it is possible, find the minimum value of KK for which it is possible.
Input Format

The first line of input contains an integer TT denoting the number of test cases. TT test cases follow.
The first line of each test case contains the integer NN - the size of arrays AA and BB.
The second line contains NN integers A1,A2,…,ANA1,A2,…,AN - the elements of the array AA.
The third line contains NN integers B1,B2,…,BNB1,B2,…,BN - the elements of the array BB.

Output Format
For each test case, output in a single line, the minimum possible value of KK. If it is not possible to convert BB to AA, print -1.
Constraints

1≤T≤2⋅1051≤T≤2⋅105 
1≤N≤2⋅1051≤N≤2⋅105
1≤Ai,Bi≤1091≤Ai,Bi≤109
Sum of NN across all test cases does not exceed 2⋅1052⋅105.


                        Sample Input 1
                        





2
4
1 4 5 2
4 5 1 2
6
1 2 3 4 5 6
1 2 5 6 3 4



                        Sample Output 1
                        





1
-1


Explanation
Test case 11: Let R=3R=3 and K=1K=1. Thus, we apply 11 right-rotation on the subarray [4,5,1][4,5,1]. The rotated subarray is [1,4,5][1,4,5]. The updated array is [1,4,5,2][1,4,5,2] which is equal to the array AA.
It can be shown that the array BB cannot be converted into the array AA using KK less than 11.
Test case 22: The array BB cannot be converted to the array AA using the mentioned operation exactly once.
